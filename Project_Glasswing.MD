# Introduction

Project Glasswing is one of the most significant AI deployment experiments conducted by Anthropic. Unlike traditional model launches, where frontier AI systems are immediately released to the public, Glasswing was designed around a different principle: advanced defensive capabilities should reach defenders before attackers.

Anthropic launched Project Glasswing in April 2026 after concluding that its next-generation Mythos-class systems had crossed an important capability threshold. The company believed that these systems were capable of discovering and exploiting software vulnerabilities at a level that created meaningful national-security and cybersecurity implications.

Rather than immediately releasing these capabilities broadly, Anthropic created a trusted-access program that provided carefully selected organizations with access to Mythos Preview, the precursor to Mythos 5.

Project Glasswing therefore serves as both a cybersecurity initiative and a policy experiment in responsible frontier AI deployment.

---

# What is Project Glasswing?

Project Glasswing is Anthropic's controlled-access cybersecurity program for frontier AI systems.

The project was announced on April 7, 2026 and centered around providing approved organizations with access to Claude Mythos Preview, a model that would later evolve into Mythos 5.

The goal was straightforward:

```text
Advanced AI Capability
           ↓
Provide Access To Defenders
           ↓
Find Vulnerabilities
           ↓
Patch Systems
           ↓
Reduce Global Cyber Risk
```

Unlike traditional beta programs, Glasswing focused specifically on organizations that maintain critical software, infrastructure, and services used by millions of people.

Anthropic described the initiative as an effort to ensure that the benefits of frontier AI capabilities reached defenders before adversaries.

---

# Why Anthropic Created Glasswing

Anthropic argues that AI capabilities have reached a point where software security can be transformed.

Historically, vulnerability discovery has been constrained by human expertise. Security researchers manually review code, identify weaknesses, and coordinate fixes. This process is slow, expensive, and difficult to scale.

Mythos Preview changed that equation.

Anthropic reported that the model demonstrated unusually strong performance in:

* Vulnerability discovery
* Software security analysis
* Penetration testing assistance
* Exploit-chain reasoning
* Security review workflows
* Large-scale code analysis

These capabilities raised an important question:

If AI can dramatically accelerate vulnerability discovery, who should gain access first?

Anthropic's answer was defenders.

---

# The Cybersecurity Capability Threshold

Project Glasswing exists because Anthropic concluded that Mythos-class systems crossed a cybersecurity threshold.

The company believed that these models were capable of discovering software vulnerabilities at a scale and speed that could significantly alter the balance between attackers and defenders.

This threshold was not simply about answering cybersecurity questions.

It involved:

* Multi-step reasoning
* Long-horizon attack planning
* Software repository analysis
* Vulnerability chaining
* Exploit-path discovery

As a result, Anthropic chose a controlled deployment strategy rather than immediate public release.

# Mythos Preview and the Origins of Glasswing

Mythos Preview served as the first deployment vehicle for Mythos-class capabilities.

The model was never intended for general public access.

Instead, it was distributed through Glasswing to selected organizations capable of applying the technology for defensive cybersecurity purposes.

The Preview phase allowed Anthropic to:

* Measure real-world effectiveness
* Gather operational feedback
* Study misuse risks
* Improve safety systems
* Validate trusted-access controls

This controlled release eventually informed the development of both Mythos 5 and Fable 5.

# Trusted Access Program Requirements

Anthropic has not published a complete application checklist.

However, public statements indicate that participating organizations generally must:

## Operate Critical Infrastructure

The organization should provide services that affect large populations or essential systems.

## Have a Defensive Mission

The primary objective must be cybersecurity defense rather than offensive operations.

## Meet Security Standards

Organizations must demonstrate responsible operational practices.

## Participate in Remediation

The value of Glasswing comes not only from finding vulnerabilities but from fixing them.

## Accept Operational Controls

Participants operate under restrictions governing the use of Mythos-class systems.

# Why Anthropic Chose Controlled Deployment

Anthropic believed that immediate public release would create unacceptable risks.

Their concerns included:

* Offensive cybersecurity misuse
* Exploit automation
* Vulnerability weaponization
* Rapid scaling of attacks

By limiting access to trusted defenders, Anthropic attempted to create a temporary advantage for those responsible for protecting systems.

This approach differs significantly from the traditional AI release model and may influence how future frontier systems are deployed.

# Conclusion

Project Glasswing represents one of the most ambitious attempts to deploy frontier AI capabilities responsibly.

Since April 2026, the initiative has demonstrated that advanced AI systems can significantly improve defensive cybersecurity operations when placed in the hands of trusted organizations.

By identifying more than 10,000 high-severity vulnerabilities and expanding to over 150 organizations worldwide, Glasswing has provided early evidence that AI can create a meaningful defensive advantage.

Whether this deployment model becomes the norm for future frontier systems remains uncertain, but Project Glasswing will likely be remembered as one of the first large-scale experiments in trusted-access AI governance.
